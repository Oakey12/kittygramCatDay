Kittygram: Кот дня

REST API сервис для публикации любимых питомцев, участия в рейтингах и голосования за лучших котов.
Проект является расширенной версией базового приложения Kittygram, дополненной соревновательной механикой и автоматическим формированием топов.

Возможности (Функционал)

Помимо базового CRUD для котов и пород, в проекте реализован пользовательский сценарий «Кот дня (витрина)»:

Кот дня: Автоматическая генерация и отображение случайного питомца на главной витрине.

Система голосования: Пользователи могут голосовать за понравившихся котов.

Защита от накруток: Нельзя голосовать за собственного кота.

Лимиты: Разрешен только 1 голос за конкретного кота в день от одного пользователя.

Недельный топ: Автоматическое формирование рейтинга (Топ-10) котов за последние 7 дней на основе полученных баллов (score).

История голосов: Авторизованные пользователи могут просматривать список котов, за которых они отдали свой голос.

🛠 Стек технологий

Язык: Python 3.11+

Фреймворк: Django, Django REST Framework (DRF)

База данных: PostgreSQL

Аутентификация: Djoser (JWT токены)

Контейнеризация: Docker, Docker Compose

Веб-сервер: Nginx, Gunicorn

Документация: drf-spectacular (Swagger UI / ReDoc)

Инструкция по локальному запуску

Проект полностью докеризирован. Для запуска вам потребуется установленный Docker и плагин Docker Compose.

1. Клонирование репозитория

Склонируйте проект на свой компьютер и перейдите в папку с проектом:

git clone <ссылка_на_ваш_репозиторий>
cd kittygram_project


2. Переменные окружения

Создайте файл .env в корневой директории проекта (рядом с docker-compose.yml) и добавьте в него следующие настройки:

SECRET_KEY=your_secret_django_key_here
DEBUG=False
ALLOWED_HOSTS=127.0.0.1,localhost,backend
DB_ENGINE=django.db.backends.postgresql
POSTGRES_DB=kittygram
POSTGRES_USER=kitty
POSTGRES_PASSWORD=kitty_pass
DB_HOST=db
DB_PORT=5432


3. Сборка и запуск контейнеров

Запустите сборку образов и старт контейнеров в фоновом режиме:

docker compose up -d --build


4. Подготовка базы данных

Выполните миграции, соберите статические файлы и создайте суперпользователя:

# Применение миграций
docker compose exec backend python manage.py migrate

# Сбор статики (для корректного отображения админки и Swagger)
docker compose exec backend python manage.py collectstatic --no-input

# Создание администратора (потребуется ввести логин, email и пароль)
docker compose exec backend python manage.py createsuperuser


Документация и доступные адреса

После запуска проект будет доступен по следующим адресам:

Интерактивная документация API (Swagger): http://localhost:8000/api/schema/swagger-ui/

Документация ReDoc: http://localhost:8000/api/schema/redoc/

Панель администратора: http://localhost:8000/admin/ (В админ-панели настроен удобный фильтр и вывод дневных топов, голосов и котов дня).
